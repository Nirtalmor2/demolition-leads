# העלאה ל-Vercel — מדריך פשוט (ללא GitHub)

המערכת מוכנה לעלייה. צריך שני דברים שדורשים חשבון אישי שלך:
**(א)** מסד נתונים חינמי בענן, **(ב)** חשבון Vercel.
כל השאר זה העתק-הדבק של פקודות בטרמינל.

> כל הפקודות מורצות בתיקיית הפרויקט (`C:\ewp\nir\niro`).

---

## שלב 1 — מסד נתונים חינמי (Neon)
1. היכנס ל-https://neon.tech → הירשם (חינם, אפשר עם Google).
2. צור פרויקט (Create Project). בחר אזור קרוב (Europe).
3. בעמוד **Connection string** העתק את הכתובת שמתחילה ב-`postgresql://...`
   (ודא שהיא כוללת `?sslmode=require`).

## שלב 2 — חיבור המסד והעלאת הטבלאות
1. פתח את הקובץ `.env` בפרויקט והדבק את אותה הכתובת גם ב-Vercel:
   ```
   DATABASE_URL="postgresql://...הכתובת-מ-Neon..."
   ```
   אם אתה משתמש ב-Neon או ב-Postgres אחר, השתמש באותה הכתובת גם בסביבה של Vercel.
2. צור את הטבלאות במסד (פעם אחת):
   ```
   npm run db:deploy
   ```
3. משוך נתונים אמיתיים לראשונה (כ-40 שניות):
   ```
   npm run ingest
   ```
   זהו — המסד מלא בלידים מכל הארץ.

## שלב 3 — העלאה ל-Vercel (דרך הטרמינל, בלי GitHub)
1. התקן את כלי Vercel (פעם אחת):
   ```
   npm i -g vercel
   ```
2. התחבר (ייפתח דפדפן לאישור):
   ```
   vercel login
   ```
3. העלאה ראשונה (ענה על השאלות עם Enter; שם פרויקט: niro):
   ```
   vercel
   ```
4. הגדר את כתובת המסד גם ב-Vercel (הדבק את אותה כתובת מ-Neon כשיתבקש):
   ```
   vercel env add DATABASE_URL production
   ```
5. העלאה לאוויר (פרודקשן):
   ```
   vercel --prod
   ```
   בסיום תקבל כתובת אתר חיה (כמו `https://niro.vercel.app`). זהו!

## עדכון יומי אוטומטי
- כבר מוגדר ב-[`vercel.json`](vercel.json) — Vercel ירענן את הנתונים כל יום ב-03:00.
- ה-cron הזה דורש מסלול **Vercel Pro**. במסלול החינמי הוא עלול להיקטע.
- **חלופה פשוטה וחינמית:** בכל פעם שתרצה לרענן נתונים, הרץ מהמחשב שלך:
  ```
  npm run ingest
  ```
  זה מעדכן ישירות את המסד בענן, והאתר יציג את העדכון מיד.

## הערות
- אם תרצה להגן על הרצת ה-cron בסוד: הוסף `CRON_SECRET` גם ב-`.env` וגם ב-Vercel
  (`vercel env add CRON_SECRET production`).
- לפיתוח מקומי ללא ענן: יש `docker compose up -d` שמרים Postgres מקומי (ראו `docker-compose.yml`).
- העלאות עתידיות של שינויים: פשוט הרץ שוב `vercel --prod`.
